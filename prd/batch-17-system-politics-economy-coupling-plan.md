# Batch 17 Implementation Plan — System Politics Engine + Economy Coupling

Date: 2026-03-04  
Owner: Product + Backend + Frontend

## Objective

- Add deterministic per-system political state progression tied to faction
  control, security, and stability.
- Couple political state to economy micro/macro tick outcomes without
  destabilizing core market loops.
- Surface political and economy impact clearly in mission generation,
  reputation, market summaries, and system-level UI.
- Deliver safe LiveOps controls (tuning, simulation, rollback guardrails)
  for this coupling layer.

## Why This Batch Next

- Batch 16 convoy expansion increases inter-system goods movement and now
  needs world-state drivers (politics/security) to stay meaningful.
- Current economy behavior is event-capable but not yet tied to persistent
  political progression, limiting mission and faction coherence.
- This batch establishes the shared state foundation required by Batch 18
  station narrative + mission depth.

## In Scope

### 1) Political State Engine (Deterministic)
- Track per-system `control_faction_id`, `stability_score`,
  `security_level`, and current political pressure signals.
- Apply deterministic political macro tick transitions with bounded delta
  rules and explicit reasons.
- Persist political tick version IDs and event records for replay/debug.

### 2) Politics-to-Economy Coupling (Tick-Aware)
- Integrate political signals into economy pressure inputs (event modifiers,
  route reliability impact, replenishment confidence impact).
- Respect existing economic controls from
  `economic-system-design.md`:
  - staged ripple propagation with decay
  - per-commodity volatility clamps
  - smoothing before publish
- Keep coupling bounded by policy: no uncapped one-tick jumps due only to
  politics state changes.

### 3) Missions and Reputation Integration
- Feed political state/security into mission generation weighting and
  mission risk/reward context.
- Ensure faction reputation effects are consistent with controlling faction
  and recent political events.
- Expose additive mission metadata for political context without breaking
  mission lifecycle APIs.

### 4) Admin and LiveOps Controls
- Add controlled tuning for politics/economy coupling parameters with
  role-based access.
- Require staged apply workflow for high-impact changes:
  stage -> approve -> activate.
- Enforce audit logs for tuning actions (actor, old/new values, reason,
  expiry).
- Add simulation/dry-run path for high-impact changes before live apply.

### 5) Player-Facing UX Surface
- Display political pressure/security effects in system and market summaries
  with explicit loading, empty, error, success states.
- Show event severity and trend cues for market shifts caused by politics.
- Provide mission context indicators tied to politics/economy state.

### 6) Validation
- Deterministic backend replay tests for politics and economy coupling.
- Contract tests for additive fields and unchanged legacy fields.
- Frontend tests for indicator rendering and status-state behavior.

## Out of Scope (Explicit)

- Deep diplomacy negotiation UI or treaty systems.
- Player-governed governments/elections.
- Full faction war simulation beyond bounded event modifiers.
- Replacing existing economy core with a new pricing model in this batch.

## Acceptance Criteria

- [ ] Political state persists and progresses deterministically by tick
      version across repeated replay runs.
- [ ] Politics-driven economy effects are bounded by configured
      clamp/smoothing rules and do not cause runaway volatility.
- [ ] Mission generation and reputation outcomes include additive political
      context without breaking existing mission APIs.
- [ ] Admin tuning changes support staged apply, audit logging, and rollback
      on guardrail breach.
- [ ] Player-facing UI surfaces political/economy effects with explicit
      loading, empty, error, and success states.
- [ ] SFX events fire with defined cooldown/channel behavior and respect
      reduced-audio settings.

## Risks and Mitigations

- Risk: runaway instability loops between politics and pricing.
  - Mitigation: bounded deltas, volatility clamps, staged ripple with decay.
- Risk: opaque economy behavior for players.
  - Mitigation: trend/severity indicators and mission context explanations.
- Risk: over-tuning during live events.
  - Mitigation: simulation gate + staged rollout + auto-rollback policy.
- Risk: shard/tick performance pressure.
  - Mitigation: telemetry thresholds, delta-first repricing, scale triggers.
